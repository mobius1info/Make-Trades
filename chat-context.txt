## Remaining limitations / next recommendations
- Static SSG output is generated in `dist` by the build; deployment must run `npm run build` or publish the regenerated `dist` output. Deploying the old static files would keep old SEO issues.
- Legacy `/blog-post.html?slug=...&lang=...` is a client fallback and generic server response. It is intentionally `noindex, follow`; sitemap/blog cards/canonicals now promote `/blog/{lang}/{slug}/`.
- Main homepage language variants still use `?lang=...`; this implementation does not generate `/en/`, `/de/`, `/uk/`, `/zh/` home pages. They are omitted from sitemap to avoid canonical/query conflicts.
- Article hreflang is conservative: full clusters are emitted only where slugs have a recognizable `-{lang}` sibling pattern; otherwise page gets self + x-default. This avoids fake hreflang clusters.
- Hidden SEO articles are not put into user-facing blog indexes; they are discoverable via static article pages and sitemap.
- After deployment: submit the new sitemap in Google Search Console / Yandex Webmaster and request recrawl for `/`, `/blog/ru/`, `/faq/ru/` and priority `/blog/{lang}/{slug}/` articles.
