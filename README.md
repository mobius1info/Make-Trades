# MakeTrades Website

Современный веб-сайт для платформы создания брокеров, криптобирж и дилинговых центров.

## Возможности

- **SEO-оптимизация**: Полный набор meta-тегов, Open Graph, Schema.org
- **Многоязычность**: Поддержка 5 языков (русский, английский, немецкий, украинский, китайский)
- **Динамический контент**: 500+ статей блога и FAQ из Supabase
- **Автоматическая публикация**: Система автоматической публикации новых статей каждый день
- **Отзывчивый дизайн**: Адаптивная верстка для всех устройств
- **Современный стек**: Vite + TypeScript + Supabase + Edge Functions
- **Админ-панель**: Полное управление контентом, локализацией и картинками

## База данных

Проект использует Supabase со следующими таблицами:

- `blog_posts` - статьи блога (500+ статей на 5 языках)
- `faq_items` - вопросы-ответы (15 вопросов на каждом языке)
- `contact_submissions` - обращения через форму
- `demo_requests` - запросы демо-аккаунтов

- **Русский (RU)**: 100 статей (10 опубликовано, 90 по расписанию)
- **Английский (EN)**: 100 статей (10 опубликовано, 90 по расписанию)
- **Немецкий (DE)**: 100 статей (10 опубликовано, 90 по расписанию)

#### FAQ

- Категории: general, technical, pricing, support

## Автоматическая публикация

Система автоматически публикует новые статьи блога каждый день:

### Edge Function: auto-publish-posts

Расположение: `supabase/functions/auto-publish-posts/index.ts`

**Что делает:**
- Проверяет все статьи с датой публикации <= текущей даты
- Автоматически меняет статус `published` с `false` на `true`
- Возвращает статистику опубликованных статей по языкам

**Как использовать:**

Вызов функции вручную:
```bash
curl -X POST https://your-project.supabase.co/functions/v1/auto-publish-posts \
  -H "Authorization: Bearer YOUR_ANON_KEY"
```

**Рекомендуется настроить cron job:**
- Вызывать функцию ежедневно в 00:00
- Использовать внешний cron-сервис (например, cron-job.org, EasyCron)
- Или настроить GitHub Actions для автоматического вызова

**Пример GitHub Actions workflow:**
```yaml
name: Auto-publish blog posts
on:
  schedule:
    - cron: '0 0 * * *'  # Каждый день в полночь UTC
  workflow_dispatch:

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - name: Call Edge Function
        run: |
          curl -X POST ${{ secrets.SUPABASE_URL }}/functions/v1/auto-publish-posts \
            -H "Authorization: Bearer ${{ secrets.SUPABASE_ANON_KEY }}"
```

## Админ-панель

Полнофункциональная админ-панель для управления всем контентом сайта.

### Доступ к админке

URL: `/admin.html`

### Учетные данные администратора

Первый администратор уже создан автоматически:

- **Email:** admin@maketrades.space
- **Пароль:** Admin123456!

**ВАЖНО:** Смените пароль после первого входа!

### Возможности админки

#### 1. Управление локализацией
- Добавление/редактирование/удаление переводов
- Поддержка всех 5 языков (RU, EN, DE, UK, ZH)
- Категоризация переводов (кнопки, цены, метки, общее)
- Поиск и фильтрация по языку и категории
- **ВСЕ кнопки на сайте используют переводы из базы данных**

#### 2. Управление картинками
- Добавление/редактирование/удаление изображений
- URL изображений и alt-тексты
- Категоризация (hero, features, icons, general)
- Поиск по ключу и категории

#### 3. Управление статьями блога
- Добавление/редактирование/удаление статей
- Управление публикацией (опубликовано/черновик)
- Настройка даты публикации
- Поддержка всех языков
- Редактирование контента в HTML
- Поиск и фильтрация по языку и статусу публикации

### Безопасность

- Row Level Security (RLS) на всех таблицах
- Только авторизованные администраторы могут изменять данные
- Все пользователи могут читать переводы и картинки
- Защита от несанкционированного доступа

### Таблицы базы данных

1. **translations** - все переводы интерфейса
2. **site_images** - все изображения сайта
3. **admin_users** - список администраторов
4. **blog_posts** - статьи блога (уже существующая)
